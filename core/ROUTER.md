# 任务路由规则（ROUTER.md）

> 定义"什么任务由哪个系统处理"。路由规则要短、清楚、可判断。

---

## WorkBuddy 负责

- 微信语音一句话触发的任务
- Notion 记账、勾选、状态更新
- 本地文件整理、汇总、输出成品
- 短任务、多次触发、高频执行类工作
- 已经定义清楚输入输出的工作流
- docx/xlsx/pdf 等文件处理

## Hermes 负责

- 长时间连续对话
- 日常提醒、固定自动化、cron
- 深度经营推演
- 长篇文档复盘与整合
- 复杂项目拆解
- 跨多轮的持续任务推进
- 需要自有 API 套餐稳定运行的思考型任务
- 24小时在线值班

## 切换规则

出现以下任一情况，相关任务直接切到 Hermes：
- WorkBuddy 自定义 API 模型频繁中断
- WorkBuddy 对话长度明显受限
- 关键任务进行到一半被截断
- 同一类长任务连续两次失败
- Mac 合盖/锁屏导致离线

## 前台原则

- 前台尽量只有一个主人格：峰哥
- 后台拆成多个 skill
- 不让用户频繁自己判断该找哪个角色
