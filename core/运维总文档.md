## 一、快速恢复（Hermes挂了怎么办）

| 场景 | 操作 |
|------|------|
| Gateway进程死了 | `hermes start` |
| Hermes完全无响应 | 重启WSL2：PowerShell `wsl --shutdown` → 重新打开 |
| 飞书收不到消息 | 检查 `hermes config get gateway.app_id`，确认飞书bot配置 |
| 模型限流 | 自动fallback回dashscope/qwen3.6-flash，无需手动干预 |
| 模型彻底不可用 | 飞书发 `/model` 切换 |

**详细文档**：`core/AI-OS重启恢复手册.md`

## 五、问题排查记录

**主文档**：`core/Hermes使用过程问题总结.md`

| 问题 | 根因 | 解决方案 |
|------|------|---------|
| SSL EOF Error | WSL2网络波动 | 重试即可 |
| Chrome v20 cookies无法解密 | Chrome 127+ app-bound encryption | 只能从Chrome进程内部解密 |
| yt-dlp抖音下载失败 | 需要登录cookies | CDP方案或cookies.txt |
| ffmpeg单行文本截断 | read_file工具限制 | 用Python `open().read()` |
| Notion API返回空属性 | API版本问题 | 必须用 `2022-06-28` |
| Cronjob schedule='now'无效 | 格式不支持 | 用 `schedule='1m'` |
| delegate_task大内容超时 | 600s限制 | 改用cronjob |
