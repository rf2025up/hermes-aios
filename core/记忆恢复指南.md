# 记忆恢复指南

> **创建**：2026-05-02 | **更新**：2026-05-02
> 如果Hermes或OpenClaw被重建/清空，按以下步骤恢复记忆

## 备份位置

所有备份在 `~/.hermes/backups/`，**在git仓库外，其他agent碰不到**。

## 备份范围

| 来源 | 文件 | 说明 |
|------|------|------|
| Hermes核心 | memory_store.db / state.db / config.yaml | fact_store + 会话索引 + 配置 |
| OpenClaw main | workspace/MEMORY.md + SOUL.md + USER.md + IDENTITY.md | 主agent + bridge共享 |
| OpenClaw edu | workspace-edu/MEMORY.md + SOUL.md + USER.md + IDENTITY.md | 小园老师 |
| OpenClaw ops | workspace-ops/MEMORY.md + SOUL.md + USER.md + IDENTITY.md | 贾维斯 |
| OpenClaw kitchen | workspace-kitchen/MEMORY.md + SOUL.md + USER.md + IDENTITY.md | |
| memos.db | memos.db | MemOS统一记忆层（激活后） |
| OpenClaw配置 | openclaw.json | 所有agent配置 |

## 恢复步骤

```bash
# 1. 确认备份存在
ls ~/.hermes/backups/

# 2. 恢复Hermes记忆
cp ~/.hermes/backups/memory_store.db.latest ~/.hermes/memory_store.db
cp ~/.hermes/backups/state.db.latest ~/.hermes/state.db
cp ~/.hermes/backups/config.yaml.latest ~/.hermes/config.yaml

# 3. 恢复OpenClaw记忆（如果也被清空）
for ws in workspace workspace-edu workspace-ops workspace-kitchen; do
  for f in MEMORY.md SOUL.md USER.md IDENTITY.md; do
    src=~/.hermes/backups/openclaw_${ws}_${f}.latest
    if [ -f "$src" ]; then
      cp "$src" ~/.openclaw/$ws/$f
    fi
  done
done

# 4. 恢复memos.db
cp ~/.hermes/backups/memos.db.latest ~/.openclaw/memos-local/memos.db

# 5. 重启
hermes gateway restart
openclaw gateway restart  # 如果OpenClaw也重建了
```

## 自动备份

- 脚本：`~/.hermes/scripts/hermes-memory-backup.sh`
- Cron：每天20:00自动执行（job: e90716fde2ab）
- 保留30天历史 + 一份 .latest

## 注意事项

- 恢复config.yaml时如果版本差异大，手动合并而非直接覆盖
- openclaw.json同理，谨慎覆盖
