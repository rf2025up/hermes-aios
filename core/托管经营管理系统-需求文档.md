## 一、项目概述

托管经营管理系统 = **虚拟CEO经营操作系统**。

核心理念：从全年经营目标出发，逐层拆解到每月→每周→每个关键环节→每个领域，形成自动化工作周报+进度追踪+数据复盘闭环，让锋哥只管执行和决策，AI负责提醒、追踪、汇总。

**定位类比：** AI Agent 作为经营大盘的总指挥，锋哥是CEO。

## 二、已落地模块

### 2.1 日常记账（双写）✅

- **触发:** 锋哥发送"买菜103"、"交电费300"等包含金额的信息
- **执行:** 同时写入 Notion 数据库（粗账） + 飞书多维表格（细账，以飞书为准）
- **飞书分类（六象限）:** 厨房 / 工资 / 房租 / 折旧 / 杂费 / 营销
- **飞书营销类型（仅营销类）:** 项目营销（跟具体项目走） / 全年营销（日常性宣发、礼品、地推）
- **杂费范围:** 办公用品、日用品、纸、水杯等日常用品
- **Notion:** 保持现有分类不变，记粗账
- **确认:** 写入后返回确认：项目/金额/分类/日期

### 2.2 学生缴费管理 ✅

- **核心问题:** 按学期缴费和按月缴费容易忘记，需要系统追踪
- **数据表:** 飞书多维表格「学生缴费管理」
- **字段:** 学生姓名 / 缴费类型(按学期|按月) / 月费用 / 缴费周期 / 已缴金额 / 待缴金额 / 下次缴费日期 / 缴费状态(已缴|待缴|逾期) / 备注
- **工作流:**
  1. 锋哥告知缴费情况 → AI更新缴费表
  2. 每月定期推送待缴费名单给锋哥

### 2.3 周汇总（待设定时任务）

- **频率:** 每周一推送
- **内容:** 上周支出按分类汇总 + 收入 + 周净额

### 2.4 月汇总（待设定时任务）

- **频率:** 每月1号推送
- **内容:** 上月支出按分类汇总 + 收入 + 月净利润 + 环比上月

### 3.2 进度看板（工作计划+问题跟进）

**需求:**
- 工作计划进度追踪：哪些已安排、推进到哪一步

### 3.4 家长跟进机制（月度深度沟通）

**核心规则:** 每个学生每月至少1次深度沟通

**需求:**
- 追踪每位学生的「本月是否已沟通」
- 未沟通的提前提醒
- 沟通后记录要点（需求变化、新问题、满意度）
- 纳入月度工作考核

播放量/曝光 → 加微信量 → 沟通量 → 到店量 → 成交量

**需求:**

- 转化活动：从接触到成交的全链路追踪
