## 一、核心问题：我在经历什么？

### 1.1 内耗的根源

我经常处于这样的状态：
- 脑子里有很多零散想法、概念、疑问
- 这些东西没有及时被清理和结构化
- 它们堆积在一起，产生内耗
- 占用心理带宽，影响执行力

**这不是"想太多了"，而是缺乏一个稳定的外部认知处理器。**

### 1.2 我真正需要的

不是一个学习系统、不是一套工具链，而是一个**与AI协作的固定模式**：

> 当我有零散想法时 → 跟AI聊透理清 → AI帮我结构化抽象推演 → 该落地的进待办清单 → 该沉淀的存档 → 心里不堵了

这个模式我已经经历过多次，在AI Studio的400+对话中反复出现，但我一直没有把它提炼成明确的协作协议。

### 1.3 两种路径的标本

我在与AI对话中，其实一直在走两条不同的路径：

| | 路径A：深度消化（好） | 路径B：高级整理（陷阱） |
|---|---|---|
| **特征** | 从混乱推演到可落地的结论 | 从具体问题推演到抽象框架 |
| **终点** | 一张能指导行动的图/话术/方案 | 一套看起来很高级的思维模型 |
| **锚点** | 有外部收敛锚点（要给家长看、要用） | 没有锚点，纯概念推导 |
| **例子** | 晚辅卖点分层图（从一段混乱的流量讨论，一路推演到可落地的卖点分层） | 战略剃刀模型（从"帮我做计划"一路推演到"四层思维白皮书"） |

**关键区别**：路径A有"收敛锚点"——它是为某个具体场景服务的。路径B没有锚点，所以永远不会自然停止。

## 二、与AI协作的双模式

### 2.1 锋刀模式（逼落地）

**用于**：业务决策、执行卡点、时间紧迫
**AI怎么做**：直击本质，拦截逃避，逼落地

### 2.2 智库模式（帮理清）

**用于**：我主动想展开一个话题、有零散想法想理清、学完东西想消化
**AI怎么做**：专业切入，深度碰撞，帮我结构化，不急着收

### 2.3 切换信号

| 我说的 | AI该用的模式 |
|---|---|
| "来做"、"帮我落地" | 锋刀 |
| "来聊"、"探讨一下"、"帮我想清楚" | 智库 |
| 拿不准时 | AI问我："这个你想聊透还是直接给结论？" |

### 2.4 智库模式的收尾流程

这是关键——智库模式不能"聊完就散"，必须有收尾：

1. **理清**：把零散想法变成清晰的概念和结论
2. **分类**：哪些是认知资产（存档），哪些是行动项（进TODO）
3. **落地**：行动项写入待办清单
4. **清空**：让我确认"这个想清楚了"，不再内耗

## 三、认知清空的完整链路

```
零散想法堵在心里
    ↓
跟AI聊（智库模式）
    ↓
概念理清：AI帮我定义概念、建立关联、消除模糊
    ↓
结构化抽象：从零散到有逻辑、从具体到可迁移
    ↓
收敛判断：
  ├── 有具体场景要用的 → 落地成方案/话术/图 → 进TODO
  └── 纯认知提升的 → 存档为知识资产 → 可随时检索
    ↓
心里清空，不再内耗
```

### 3.1 收敛的判断标准

怎么判断一个讨论"可以停了"？答案是看有没有**外部锚点**：

| 有锚点（可以收敛） | 无锚点（继续聊或暂停） |
|---|---|
| 这个结论明天要用来跟家长讲 | 想把框架再打磨得更完美 |
| 这周就要执行了 | 觉得还有一层没想透 |
| 有具体的人会用到这个产出 | 没有人在使用，纯自我提升 |
| 停下来的产出是"够用版" | 停下来的产出是"又多了一个框架" |

**简单说：如果有东西要交出去（给客户、给老师、给自己执行），就够用了。如果没有，那个框架大概率是路径B。**

## 四、TOPICS与TODO的关系设计

### 4.1 核心原则

```
TOPICS = 我在想什么（认知层面）
TODO = 我要做什么（执行层面）
```

两者不是独立的，而是一个**漏斗**：

```
零散想法 → 捕获进TOPICS → 聊透理清 → 拆出行动项进TODO → 完成后结论沉淀回TOPICS/MEMORY
```

### 4.2 议题的生命周期

| 阶段 | 状态 | 含义 |
|------|------|------|
| 捕获 | 🔴 想法刚冒出来 | 还模糊，需要理清 |
| 探讨 | ⚙️ 在跟AI讨论中 | 正在结构化 |
| 成熟 | ✅ 理清了，有结论 | 认知资产，可检索 |
| 转化 | → 进TODO | 拆出了具体行动项 |
| 沉淀 | 📦 入MEMORY | 长期知识资产 |

### 4.3 什么时候从TOPICS拆任务到TODO

- 想清楚"要做什么"的时候（不再是"在想什么"）
- 有明确的下一步动作
- 有时间节点

**什么时候不拆**：还在理清阶段、还没找到具体动作、纯粹是概念理解

### 4.4 AI的协作职责

| 环节 | AI做什么 |
|------|---------|
| 捕获 | 自动从对话中识别零散想法，提醒是否要记录到TOPICS |
| 探讨 | 智库模式下帮理清概念、结构化推演 |
| 成熟判断 | 判断讨论是否可以收敛，提出收敛建议 |
| 拆任务 | 识别讨论中可以变成行动项的部分，征求我同意后写入TODO |
| 存档 | 结论和认知资产整理好，存入TOPICS或独立文档 |

## 五、实际操作协议

### 5.1 当我有一个零散想法时

1. 我在飞书/语音发给AI（一句话也行，一段话也行）
2. AI判断这是"要聊透"还是"要落地"
3. 如果是"要聊透"→ 进入智库模式，帮理清
4. 聊完后AI主动做收尾：
   - "结论是这样的……"
   - "这个可以拆出以下行动项：1. xxx 2. xxx"
   - "要不要进TODO？"
   - "认知部分我存一份到TOPICS"

### 5.2 当我在学习一个课程/视频/文章时

1. 学完后把核心概念发给AI
2. AI用提问的方式帮我检验理解深度（问我，不是教我）
3. 我不会的AI再教，反复几轮
4. 最后AI帮我沉淀：
   - 核心概念卡片（是什么、为什么重要、怎么用）
   - 可直接行动的部分 → 进TODO
   - 认知提升的部分 → 存档

### 5.3 当我脑子里堵了很多东西时

1. 随便倒出来，不需要有逻辑
2. AI帮我分类：
   - 这些东西分别属于什么领域/问题
   - 哪些其实是一个问题的不同面
   - 哪些是独立议题
3. 逐个清理，一个一个理清
4. 理清一个划掉一个，直到全部清空
