### 2.2 转写流程（已验证）

```
1. Files.upload(file_path) → 获取 file_id
2. Files.get(file_id)     → 获取 file_url（带签名的OSS临时URL）
3. Transcription.call(model='paraformer-v2', file_urls=[url]) → 提交任务
4. Transcription.wait(task) → 等待完成（通常几十秒）
5. 下载 transcription_url → JSON解析 → transcripts[0].text = 全文
```

## 四、清洗规则（核心要求）

### 4.1 直播内容（朋阳/凌霄/老司机等）— 必须清洗

**删除：**
- 直播互动（"欢迎进入直播间""X总好""来了来了"）
- 拉数据（"点赞""关注""加粉丝团""亮灯牌"）
- 引流话术（"留联系方式""后台私信""加群"）
- 拉停留（"等一下""别走""马上开始"）
- 瞎聊/闲聊/无关话题
- 重复内容、卡顿重复的话
- "能听到的打个1"等无意义互动

**保留：**
- 与主题相关的核心讲解内容
- 知识点、方法论、案例
- 问答中的有价值问题和回答
- 操作步骤、流程说明

### 4.2 短视频 — 先判断再决定

1. 先通读转写内容，判断是否有大量直播互动
2. 没有 → 直接按格式输出，不做清洗
3. 有 → 按直播清洗规则处理

### 4.3 不同主播的处理差异

| 主播 | 类型 | 处理要求 |
|------|------|----------|
| **朋阳** | 托管经营类 | 清洗后保留招生、运营、产品规划、晚辅、收心班等干货 |
| **凌霄** | 学习方法类 | **按结构化方式沉淀**，用标题层级体现知识体系（语文/数学的学习路径、方法、参考书、训练、时间规划） |
| **老司机** | 托管经营类 | 清洗后保留干货 |
| **短视频** | 各种 | 先判断是否需要清洗 |

## 七、场景B — 链接/博主主页自动下载工作流

### 7.1 触发方式
- 老板在飞书转发**抖音视频链接**（单个视频）
- 老板在飞书转发**抖音博主主页链接**（批量下载该博主全部视频）
- 老板在飞书转发**微信视频号链接**（单个视频）

### 7.2 下载工具

| 工具 | 版本 | 用途 |
|------|------|------|
| **yt-dlp** | 2026.03.17 | 下载抖音/视频号等平台视频 |
| **ffmpeg** | 已安装 | 视频转音频(.m4a, aac 128k, 16kHz, 单声道) |

### 7.3 单个视频下载流程

```
1. 收到飞书消息中的视频链接
2. yt-dlp 下载视频到临时目录
   yt-dlp -o "H:\抖音直播录制\临时下载\%(title)s_%(id)s.%(ext)s" "<链接>"
3. ffmpeg 提取音频
   ffmpeg -y -i <视频路径> -vn -acodec aac -b:a 128k -ar 16000 -ac 1 <音频路径>.m4a
4. Paraformer-v2 转写
5. AI清洗 + 生成Markdown文档
6. 文档保存到对应 _文案 目录
7. 回复老板：文档路径 + 内容概要
```

### 7.4 博主主页批量下载流程

```
1. 收到飞书消息中的博主主页链接
2. yt-dlp 列出该博主所有视频
   yt-dlp --flat-playlist --print "%(id)s %(title)s %(duration_string)s" "<主页链接>"
3. 展示视频列表给老板确认（标题+时长）
4. 老板确认后，yt-dlp 批量下载
   yt-dlp -o "H:\抖音直播录制\<博主名>\%(title)s_%(id)s.%(ext)s" "<主页链接>"
5. 批量 ffmpeg 提取音频
6. 批量 Paraformer-v2 转写
7. 批量 AI清洗 + 生成Markdown文档
8. 回复老板：完成数量 + 文档目录路径
```

### 7.5 注意事项
- 抖音链接需要能正常访问（部分私密/好友可见视频无法下载）
- yt-dlp 需要定期更新（`pip install -U yt-dlp`），平台反爬会变化
- 下载目录临时文件用完可清理，或移到对应源文件夹归档
- 视频号链接下载可能需要cookie（yt-dlp支持 `--cookies` 参数）
