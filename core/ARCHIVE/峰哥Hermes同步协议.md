# 峰哥 ↔ Hermes 同步协议

**生效日期**：2026-04-15
**版本**：v1.0

---

## 一、角色定义

| | 峰哥（WorkBuddy/Mac） | Hermes（VPS） |
|--|--|--|
| **定位** | 主力对话助手 | 24小时在线备份 |
| **强项** | 深度对话、记忆沉淀、决策支持、本地文件操作 | 定时提醒、cron任务、Notion操作、持续在线 |
| **在线时间** | Mac开机时（约早8-晚8） | 24×7 |
| **信息密度** | 拥有完整上下文（记忆库+修炼总纲+所有对话历史） | 需要同步获得上下文 |

## 二、同步内容与频率

### 1. 每日日志同步（每日1次）
- **触发时间**：每天对话结束时（约20:00-21:00）
- **内容**：当日daily日志的核心摘要
- **格式**：
  ```
  📋 【每日同步】2026-04-15
  
  ## 今日要事
  - 事件1
  - 事件2
  
  ## 关键决策
  - 决策内容
  
  ## 待办进展
  - 进展更新
  
  ## 明日重点
  - 待办事项
  ```
- **通道**：飞书群消息

### 2. 记忆文件同步（每周1次或重大更新时）
- **触发条件**：
  - 每周六统一同步一次
  - MEMORY.md有重大更新时即时同步
- **方式**：Git push到共享仓库 → Hermes自动pull
- **覆盖文件**：
  - `.workbuddy/memory/MEMORY.md`
  - `.workbuddy/memory/YYYY-MM-DD.md`（当周日志）
  - `.workbuddy/memory/议题积累.md`（议题追踪）
  - `.workbuddy/memory/知识IP素材积累库.md`（长线素材库）
  - `skills/`（新增或更新的skill）

### 3. 议题专项同步（重要！）
- **触发**：每次对话中产生新议题或议题有重大推进时
- **即时Git推送**（不等周六），确保Hermes下次推送时议题上下文完整
- **覆盖**：议题积累.md + 对应的daily日志

### 4. 重大决策即时同步（不定期）
- **触发条件**：
  - 业务方向调整
  - 战役目标/策略变更
  - 新的常驻角色或Agent提示词变更
  - 用户明确提出"告诉Hermes"
- **方式**：飞书群消息，标注 `🔴【重大同步】`
- **格式**：
  ```
  🔴【重大同步】标题
  
  ## 变更内容
  具体描述
  
  ## 影响范围
  影响到哪些日常工作
  
  ## Hermes需要调整的
  具体调整项
  ```

## 三、Hermes→峰哥的信息流

### Hermes可以主动同步给峰哥的信息：
- **定时提醒执行情况**：哪些提醒已推送、用户反馈如何
- **Notion记账记录**：当天记录了哪些账目
- **异常告警**：cron任务失败、服务异常等

### 方式：
- 通过飞书群消息，标注 `🟢【Hermes汇报】`

## 四、容灾切换协议

### 场景1：峰哥不可用（Mac关机/合盖）
1. 用户在飞书群@Hermes说明情况
2. Hermes从Git拉取最新MEMORY.md和日志
3. Hermes基于同步的上下文接手对话
4. 峰哥恢复后，Hermes汇报离线期间事项

### 场景2：Hermes不可用（VPS故障）
1. 峰哥直接告知用户
2. 定时提醒暂时通过WorkBuddy自动化（服务号推送）兜底
3. Hermes恢复后自动接回定时任务

### 场景3：两者都不可用
- 用户通过飞书/微信直接操作
- 恢复后手动对齐

## 六、Git同步规则

### 仓库：`https://github.com/rf2025up/aios.git`
### 分支策略：main分支直推（小团队无需分支）

### 峰哥的同步操作：
```bash
# 每日对话结束后
cd ~/Documents/AI-OS
git pull --rebase
# 确认MEMORY.md和日志文件是最新的
git add .
git commit -m "daily: YYYY-MM-DD 峰哥同步"
git push
```

### Hermes的同步操作：
- cron定时pull（每小时1次）
- 收到飞书群同步消息后立即pull

## 七、注意事项

1. **峰哥是信息源**：所有深度对话、决策、记忆沉淀发生在峰哥这边，Hermes是消费者
2. **不双向冲突**：Hermes不修改MEMORY.md，只读取
3. **简单优先**：同步协议先跑起来再优化，不过度设计
4. **老板在场时对齐**：重要事项在飞书群讨论时老板能看到，确保透明
