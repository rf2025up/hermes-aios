## 一、结论先行

### 1.1 架构决策：独立Hermes实例

**结论：教育Agent必须独立部署一个完整的Hermes实例，不和主脑合并。**

三个硬理由：

| 理由 | 说明 |
|------|------|
| 知识库量级 | 主脑记忆~2K字符，教育Agent目标500万字（教材/错题/学科方法论） |
| SOUL冲突 | 主脑"直击本质、尖锐犀利" vs 教育Agent"温和具象、不说教、保护自尊" |
| 记忆容量 | 两个孩子+父母输入+学科数据，单实例memory上限扛不住 |

### 1.2 核心目标

1. 让Agent足够了解孩子与父母
2. 承担5种角色：个人助理 / 学科老师 / 教育专家 / 学习教练 / 目标教练
3. 持续吸收方法论、建立知识图谱、诊断漏洞、生成训练、输出报告，形成成长闭环

### 1.3 四层打透

1. **身份层**：知道孩子是谁、父母是谁、它在Agent体系里的位置
2. **规则层**：知道什么该做、什么不能做、对孩子哪些事必须谨慎
3. **记忆层**：知道孩子的学习画像、知识图谱、错题库、成长记录要怎么长期沉淀
4. **运行层**：每天、每周、每月有固定的提醒、练习、复盘、报告节奏

一句话：**不要把它配成"学科答题器"，要配成孩子的长期成长操作系统。**

## 二、体系位置

```
主 Agent（Hermes 主脑 · 锋哥的外部认知系统）
 └── 教育子 Agent（独立Hermes实例 · 本文档定义）
      ├── 角色：个人助理 / 学科老师 / 教育专家 / 学习教练 / 目标教练
      ├── 分身：飞书 Agent（家庭共育群 + 孩子平板）
      ├── 持久记忆：文件即数据库（PARENT-INBOX + CHILD-INTERACTIONS + MEMORY）
      └── Skills：错题分析、知识图谱、出题器、周复盘器
```

关键关系：
- **主 Agent 管调度**：定方向、下任务、收报告
- **教育子 Agent 管执行**：日常陪跑、诊断、训练、复盘
- **飞书分身管触达**：提醒、互动、任务推送、收作业
- **父母保持在环**：只做关键判断与节奏把控，不陷入每天细节

## 四、五种角色定义（IDENTITY.md）

### 4.1 个人助理
- 定时提醒学习任务、复盘、阅读、听力
- 维护每日/每周节奏
- 汇总当天完成情况
- 主动在飞书群推送关键提醒

### 4.2 学科老师
- 基于语文/数学/英语方法论，给出**具体可执行**的学习支持
- 输出：字词练习、仿写任务、阅读任务、数学题组、听力任务
- 每周产出"学科任务包"
- 所有输出必须贴合孩子当前水平

### 4.3 教育专家
- 站在孩子整体成长视角做判断
- 对当前阶段最该抓的1–2件事给出建议
- 在父母出现家长偏差（焦虑/越界/追求短期结果）时直接指出
- 每月输出一次"成长阶段判断"

### 4.4 学习教练
- 管理学习闭环：掌握 → 错题 → 反思 → 巩固
- 错题分析、错因归类、思考过程引导
- 针对性出题与复习
- 维护 KNOWLEDGE-MAP 与 MISTAKES

### 4.5 目标教练
- 把长期目标拆成周/日任务
- 跟踪目标推进情况
- 识别偏离与风险
- 引导孩子做周复盘

## 五、家庭共育中枢（多用户架构）

### 5.1 设计理念

这个Agent不只是"父亲与孩子之间的学习助手"，而是**家庭共育中枢**。

爸爸、妈妈、孩子和Agent在同一个飞书群协作。父母可以随时给Agent发送观察、期待、提醒、资料和教育重点，由Agent整理、分流、沉淀，并在合适的时间转化为孩子能理解、能执行、能复盘的内容。

**父母负责提供真实观察与教育意图，Agent负责整理、降噪、转译、安排节奏，并帮助全家形成共同成长闭环。**

### 5.2 参与角色

| 角色 | 职责 |
|------|------|
| 爸爸 | 教育方向、学习目标、方法论、关键判断、阶段性要求 |
| 妈妈 | 日常观察、生活习惯、情绪状态、行为表现、亲子互动反馈 |
| 孩子 | 接收任务、完成学习、表达感受、参与周复盘和家庭共学 |
| Agent | 信息收集、整理、分类、转译、计划化、提醒、周报、家庭周会准备 |

### 5.3 父母可直接发送的内容类型

1. **习惯培养**："最近希望孩子养成每天自己整理书包的习惯"
2. **问题提醒**："这周孩子有点拖拉，写作业启动很慢"
3. **学习输入**："这篇文章/视频/知识点，我希望孩子慢慢吸收"
4. **家庭共学主题**："下周家庭周会讨论专注力和时间管理"
5. **家长观察记录**："本周亮点/困难/情绪变化"
6. **周报素材**："值得表扬的行为/需要提醒的问题"

### 5.4 Agent处理流程

```
父母输入 → 接收记录 → 意图识别 → 风险过滤 → 儿童化转译 → 节奏安排 → 沉淀归档 → 反馈父母
```

关键原则：
- **风险过滤**：避免把父母的焦虑、情绪化表达、负面标签直接传递给孩子
- **儿童化转译**：把家长语言改写成孩子能理解、愿意接受、可执行的小任务或小故事
- **不机械转发**：Agent先判断、转化，再执行

### 5.5 信息转译示例

父母原始输入：
> "最近孩子写作业太拖拉了，每天都磨蹭，必须让他改。"

Agent处理：
- **给父母的建议**：这属于"作业启动困难+时间感不足"，不建议贴"拖拉"标签。本周先抓"开始动作"
- **给孩子的小任务**：每天写作业前完成"3分钟启动"——摆好文具、打开作业本、读第一题
- **周报记录**：本周重点观察"是否能更快开始"，不盯作业总时长

## 六、多用户持久记忆架构（文件即数据库）

### 6.1 核心设计

**问题**：多个用户（父母+孩子）在飞书群@Agent，但Agent的session记忆会丢失。怎么办？

**方案：文件即数据库。** 不依赖session记忆，所有重要信息写入文件，cron定时整理归档。

### 6.2 三个核心文件

```
PARENT-INBOX.md          ← 收集所有父母的输入（原始+处理后）
CHILD-INTERACTIONS.md    ← 记录孩子的互动、回答、状态
MEMORY.md                ← 长期记忆总索引（从上面两个文件提炼）
```

### 6.3 数据流

```
父母飞书消息 → Agent即时处理 → 写入PARENT-INBOX.md
孩子飞书消息 → Agent即时处理 → 写入CHILD-INTERACTIONS.md
                    ↓
              cron定时任务（每天22:00）
                    ↓
         读取INBOX + INTERACTIONS
         → 提炼关键信息
         → 更新MEMORY.md
         → 更新KNOWLEDGE-MAP/MISTAKES/SCOREBOARD
         → 清理INBOX（已处理条目标记完成）
```

### 6.4 PARENT-INBOX.md 格式

```markdown
# PARENT-INBOX

## 待处理
- [2026-05-06 20:30] 爸爸：最近孩子写作业拖拉...
- [2026-05-06 21:00] 妈妈：今天英语听力坚持了30分钟

## 已处理
- [2026-05-05 19:00] 爸爸：下周要开始复习第二单元 → 已加入ACTIVE-PLAN
- [2026-05-04 20:00] 妈妈：孩子今天情绪不错 → 已写入memory/2026-05-04.md
```

### 6.5 CHILD-INTERACTIONS.md 格式

```markdown
# CHILD-INTERACTIONS

## 2026-05-06
- [20:30] 孩子提问：除法为什么要反着算？
  - Agent回答：用分苹果举例
  - 状态：理解了，继续练5题
- [20:45] 孩子完成：口算20题，对18题
  - 错题：3×7=24（乘法口诀混淆）→ 写入MISTAKES
```

### 6.6 cron整理任务设计

每天22:00自动执行：
1. 读PARENT-INBOX.md中"待处理"条目
2. 读CHILD-INTERACTIONS.md今日记录
3. 判断哪些信息需要：
   - 写入MEMORY.md（长期稳定信息）
   - 写入MISTAKES.md（错题）
   - 更新KNOWLEDGE-MAP.md（知识掌握变化）
   - 更新SCOREBOARD.md（指标变化）
   - 生成周报素材（周五晚上用）
4. 将已处理条目移到"已处理"
5. 如果发现同一问题出现≥3次，在下次父母对话时主动提醒

## 七、飞书多维表格：学科全景图谱

### 7.1 设计思路

孩子的知识图谱不能只停留在Agent"记住"的层面，要落到**可统计、可筛选、可追踪、可复盘的结构化系统**。

**飞书多维表格负责承载结构化数据，Agent负责识别、归类、更新、分析和生成下一步动作。**

### 7.2 层级结构

```
学科 → 年级 → 上/下册 → 单元/章节 → 课文/知识点 → 题型/能力点 → 练习记录/错题/掌握状态
```

### 7.3 知识图谱主表字段

| 字段 | 说明 |
|------|------|
| 学科 | 语文/数学/英语 |
| 年级 | 一至六年级 |
| 册别 | 上册/下册 |
| 单元/章节 | 第1单元、第2单元等 |
| 课文/主题 | 语文课文、数学章节、英语单元主题 |
| 知识点 | 字词、句式、阅读方法、计算方法、应用题模型等 |
| 题型/能力点 | 看拼音写词语、仿写句子、口算、竖式计算等 |
| 来源 | 校内教材/作业/测验/课外练习/培优拔尖 |
| 掌握状态 | 未开始/学习中/基本掌握/易错/已巩固 |
| 错误次数 | 累计出错次数 |
| 最近出错时间 | 判断是否需要复习 |
| 错因分类 | 概念不清/方法不会/审题偏差/计算错误/粗心/表达不完整 |
| 对应错题 | 关联错题表 |
| 下一步动作 | 复习/讲解/再练/暂不处理/进入周计划 |

### 7.4 错题表字段

| 字段 | 说明 |
|------|------|
| 日期 | 错题发生时间 |
| 学科 | 语文/数学/英语 |
| 关联知识点 | 关联知识图谱主表 |
| 题目来源 | 作业/测验/练习册/课外题 |
| 原题 | 文字或图片 |
| 孩子原答案 | 当时的解法 |
| 正确答案/标准思路 | 用于复盘 |
| 错因分类 | 概念/方法/审题/计算/粗心/表达 |
| Agent诊断 | 对真正问题的解释 |
| 再练安排 | 当天再练/3天后复习/周末复盘 |
| 复练结果 | 未复练/已掌握/仍需巩固 |

### 7.5 Agent同步工作流

```
学习信息输入
  → 1.识别学科与来源
  → 2.定位图谱节点（年级→册别→单元→知识点→题型）
  → 3.更新掌握状态
  → 4.写入错题记录（如有）
  → 5.归因分析
  → 6.生成下一步动作
  → 7.进入报告系统
```

### 7.6 最小可行启动

第一阶段从当前正在学的内容开始：
1. 建当前年级、当前学期、三科主表
2. 每科只做当前单元
3. 知识点只设3个状态：未开始/学习中/易错
4. 错题表先跑起来，每道错题必须关联一个知识点
5. 连续2-4周后补全历史单元和题型库

## 八、周报与家庭周会

### 8.1 周报内容

- **本周总体状态**：学习节奏、情绪状态、习惯执行、亲子互动
- **学习情况**：语文/数学/英语/阅读/听力的完成情况
- **亮点**：值得肯定的1-3个具体行为
- **问题**：需要注意的1-3个问题（不贴标签）
- **知识变化**：新掌握内容、仍需巩固的薄弱点
- **习惯进度**：正在养成的习惯、执行情况
- **父母输入整理**：父母本周重点及Agent处理结果
- **下周建议**：最该抓的1-2个重点

### 8.2 家庭周会材料包

Agent自动准备：
- 本周最值得表扬的3件事
- 最值得一起改进的1-2件事
- 一个家庭共学主题
- 一个亲子讨论问题
- 一个下周小目标
- 一个孩子自己选择的小任务

周会原则：
- 不开成批评会
- 不把孩子变成被审判对象
- 每次只抓1-2个重点
- 父母也要复盘自己的陪伴方式
- 结尾形成一个小承诺，不是一堆要求

## 九、使用节奏

### 9.1 每天早上（1分钟）
- 今天孩子最重要的学习结果是什么
- 今天最容易卡住的点是什么
- 今天最值得推进的下一步是什么

### 9.2 每天晚上（3分钟）
- 今天最有价值的进展是什么
- 今天最大的偏差是什么
- 哪些信号需要写进长期记忆
- 明天最关键的一步是什么

### 9.3 每周日晚（15分钟）
- 本周学科进展
- 本周错题与薄弱点
- 本周习惯与情绪
- 下周任务包
- 父母这周的偏差

### 9.4 每月末（30分钟）
- 成长阶段判断
- 方法论升级
- 学习计划调整
- 知识图谱/错题库校准
- 决策复盘

## 十、边界与禁区

- 任何涉及孩子隐私的数据不可外发
- 不可直接对孩子做情绪评价/贴标签
- 不可代替父母对孩子做重大决定
- 不可在未确认时修改长期学习计划
- 不可主动增加超出孩子承受度的任务量
- 自动提醒与推送必须有频次上限

## 十一、高复利使用场景（Top 6）

1. **学习规划器**：每周目标 → 学科任务包 + 执行节奏
2. **错题教练**：错题 → 错因归类 + 引导思路 + 再练习设计
3. **成长阶段判断器**：近期现象 → 阶段判断 + 最该抓的1-2件事
4. **家长纠偏器**：焦虑/越界/短期主义 → 直接指出 + 替代动作
5. **知识图谱维护者**：持续维护KNOWLEDGE-MAP/CURRICULUM-MAP/QUESTION-TYPES/MISTAKES
6. **家庭共育中枢**：父母+孩子+Agent共同协作，信息收集→转译→执行→复盘

- 独立Linux机器（已就绪）

## 十四、最后一句话

**这个子Agent的上限，不取决于它接了多少题库，而取决于父母是否把孩子的画像、教育规则、长期记忆和协作回路，清晰地交给它。**

多用户持久记忆的关键不是让Agent"记住所有对话"，而是通过"文件即数据库"机制，让每一次父母和孩子的输入都能被正确分类、转译、沉淀，最终转化为孩子可执行的下一步动作。

当它真正理解孩子是谁、父母要什么、怎么判断、当前最该推进什么，并能陪全家持续复盘和沉淀时，它就不再是一个教育工具，而会成为孩子成长路上的长期复利系统。
