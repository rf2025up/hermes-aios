## 三、转写技术方案

- **核心引擎**：阿里云百炼 DashScope Paraformer 语音识别
- **使用模型**：paraformer-v2（主力）、paraformer-v1、fun-asr-2025-11-07（备选）
- **API版本**：阿里云百炼PAAS API（`/api/coding/paas/v4/`）
- **转写流程**：音频提取（m4a）→ 格式转码 → Paraformer提交 → 结果回收 → txt保存 + JSON索引
- **脚本位置**：`/root/workspace/aios/scripts/ARCHIVE/batch_transcribe_queue.py`（已归档）
- **结果位置**：
  - 文本文件：`/root/workspace/aios/scripts/transcription_results/`
  - JSON索引：`paraformer_results_v2.json`（34MB）
  - 朋阳知识卡片：`/mnt/h/Hermes/朋阳_全年节点/`

### 遇到的技术问题及解决

| 问题 | 解决方案 |
|------|---------|
| DashScope主key欠费（报429"余额不足"） | 切换到备用key（bailian-backup-202605），实际是限流非欠费 |
| PAAS API与标准API URL混淆 | 确认使用 `/api/coding/paas/v4/` URL，标准URL会报误导性429 |
| 音频格式不兼容 | 先用ffmpeg转码为标准m4a再提交 |
| Shell中文属性名解析失败 | 转用Python subprocess调用curl，避免jq+中文冲突 |
| 长音频分段转写 | 按时间戳自动分段，生成 `_full.txt` 合并版本 |

## 七、项目总结

### 做对了什么
1. **批量自动化**：全自动转写流水线（音频提取→转码→识别→保存），无需人工干预
2. **容错设计**：多key轮换、多模型备选、断点续传，保证大规模任务不中断
3. **知识提炼**：不仅转写，还做了朋阳175个知识卡片的结构化提炼，按全年经营节点组织
4. **去重处理**：跨来源去重，朋阳和凌霄的重叠内容已识别合并

### 踩过的坑
1. **API限流vs欠费混淆**：DashScope的429错误信息"余额不足"实际是限流，浪费了排查时间
2. **API版本选择**：PAAS版和标准版URL不同，用错会报莫名其妙的错误
3. **Shell中文陷阱**：jq+中文属性名在Shell中不可靠，必须用Python处理

### 资产价值
- **1168万字**的托管行业知识文本库，覆盖招生、续费、交付、管理、薪酬、寒假/暑假/春秋季全周期
- **朋阳12个月度知识文档**可直接作为托管经营参考手册使用
- 凌霄、老司机等来源提供多元视角，可交叉验证和补充
